# stratum — 情報設計

## 公開範囲モデル

投稿には以下の公開範囲を設定できる。

```
private ── 自分のみ
グループ ── 指定した1つのグループ
複数グループ ── 指定した複数のグループ
public ── 組織全体
```

### デフォルト値の決定ルール

公開範囲のデフォルト値は以下の優先順位で決まる。

1. **個人設定**が存在すればそれを採用
2. なければ**組織設定**を採用
3. いずれも未設定の場合は **public**

## 外部公開リンクのライフサイクル

```
[発行] ──▶ [有効] ──▶ [期限切れ or 手動無効化]
                          │
                          ▼
                       [無効]
```

### 状態遷移の詳細

| イベント | 遷移 |
|----------|------|
| リンク発行 | → 有効 |
| 有効期限到達 | 有効 → 無効 |
| 投稿者が手動無効化 | 有効 → 無効 |
| 投稿者が退会 | 有効 → 無効（自動） |
| 組織管理者が機能を OFF | 有効 → 無効（組織内の全リンク） |

### 有効期限の設定

- プリセット: 1日 / 7日 / 30日 / 無期限
- カスタム: 任意の日時を指定
- 組織管理者は有効期限の上限を設定可能（例: 最大30日）

### 組織管理者の制御

- 外部公開リンク機能自体の ON/OFF
- 有効期限の上限設定（この上限を超える期限は設定不可）

## 退会時のデータ扱い

### 決定済み

| データ種別 | 扱い |
|------------|------|
| public / グループ公開の投稿 | 組織内で継続閲覧可能 |
| 外部公開リンク | 自動無効化 |

### 未決定

| データ種別 | 状態 |
|------------|------|
| private 記事 | 今後議論（削除 / 一定期間保持 / 管理者移管など） |

## 関連記事の自動提示（未決定）

投稿の Embedding（pgvector）を活用し、関連する記事を自動的に提示する構想。

### 想定される提示タイミング

| タイミング | 内容 |
|------------|------|
| 記事閲覧時 | 閲覧中の記事と Embedding が近い記事を「関連記事」として表示 |
| 記事投稿時 | 投稿内容と Embedding が近い既存記事を「関連する既存記事」として提示（重複投稿の防止効果も期待） |

### 今後議論が必要な点

- 表示件数の上限（例: 最大5件）
- 類似度の閾値（どの程度近い記事を表示するか）
- 公開範囲を考慮した絞り込み（閲覧権限のない記事は除外）
- 投稿時の提示が執筆フローを妨げないための UX 設計
